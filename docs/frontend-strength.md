# 프론트엔드 성과 정리 프레임워크 (MECE)

> **대원칙**: "무엇을 했다"가 아니라 "그게 어떤 숫자를 움직였다"가 성과다.
> 모든 항목은 `Before → After + 숫자 + 그게 만든 결과` 형태로 적는다.
> 예) "결제 플로우 리팩토링"(X, 작업) → "결제 플로우 개선으로 결제 완료율 12% → 17%, 월 결제 건수 +N건"(O, 성과)

---

## MECE 분류 축

임팩트가 **어디에 닿는가**를 기준으로 6개로 나눈다. 위에서 아래로 갈수록 "비즈니스에서 멀어지고 엔지니어링에 가까워진다."

| #   | 카테고리               | 한 줄 정의                     | 핵심 질문                       |
| --- | ---------------------- | ------------------------------ | ------------------------------- |
| 1   | 사업·매출 임팩트       | 데려온 사용자를 전환·유지·과금 | "돈/핵심 지표를 움직였나?"      |
| 2   | 유입·획득 (SEO·그로스) | 사용자를 데려옴 (퍼널 상단)    | "트래픽/신규 유저를 늘렸나?"    |
| 3   | 제품 경험 품질         | 만든 화면 자체의 질            | "빠르고, 안 깨지고, 쓰기 좋나?" |
| 4   | 데이터·계측            | 숫자를 _알게_ 만드는 인프라    | "측정 가능하게 만들었나?"       |
| 5   | 엔지니어링 시스템·DX   | 개발 효율·구조·재사용          | "다음 개발을 빠르게 했나?"      |
| 6   | 조직·협업·지식         | 사람과 팀                      | "팀의 역량을 키웠나?"           |

> 경계 규칙(겹침 방지)
>
> - **유입(2) vs 사업(1)**: 2는 "사람을 들여보내는 것", 1은 "들어온 사람에게 일어나는 경제적 변화". SEO는 2, 전환율은 1.
> - **경험 품질(3) vs 사업(1)**: 3은 "산출물의 비기능적 품질 자체", 1은 "그 품질이 만든 사업 결과". LCP 단축은 3, 그로 인한 이탈률 감소는 1. (둘을 인과로 엮으면 가장 강력한 서술)
> - **데이터(4) vs 사업(1)**: 4는 "측정 인프라를 만든 것", 1은 "측정된 숫자가 좋아진 것". GA4 이벤트 설계는 4, 전환율 개선은 1.

## 1. 사업·매출 임팩트 (가장 강력함)

프론트가 비즈니스 숫자에 직접 닿았다는 증거. 면접관·평가자가 가장 먼저 본다.

- **전환율 (CVR)**
  - 온보딩/가입 플로우 개선 → 가입 완료율 ↑
  - 결제·구독 플로우 개선 → 결제 완료율, 구독 전환율 ↑
  - 퍼널 단계별 이탈 지점 개선 → 단계 통과율 ↑
- **활성화 (Activation)**
  - 핵심 액션(첫 일정 등록, 첫 즐겨찾기 등) 완료율 ↑
  - Aha-moment 도달률 / 도달 시간 ↓
- **리텐션·이탈**
  - 핵심 화면 UX 개선 → D1 / D7 / D30 리텐션 ↑
  - 이탈률(bounce / churn) ↓
- **수익 지표**
  - ARPU / ARPPU, 결제 건수, 구독 유지율
  - 가격·플랜 노출 UI 개선 → 상위 플랜 전환 ↑

> **매핑 예시**: "구독 전환 플로우 개선 → 전환율 X%p ↑, 월 구독 매출 +₩N"
> **매핑 예시**: "온보딩 단축 → 첫 핵심 액션 도달률 X% → Y%, D7 리텐션 20% 유지"

## 2. 유입·획득 (SEO·그로스) — 강점 영역

사용자를 데려오는 퍼널 상단. 네가 실제로 가장 구체적으로 쓸 수 있는 영역.

- **기술 SEO**
  - 구조화 데이터(Event schema) 적용 → 리치 결과 노출
  - URL / slug 구조 개선 → 인덱싱·크롤링 효율 ↑
  - sitemap / canonical / robots 정비 → 중복·누락 색인 해소
  - 렌더링 전략(SSR/SSG/ISR) → 크롤러 접근성 ↑
- **콘텐츠·페이지 SEO**
  - 스케줄 페이지 개선 → 오가닉 트래픽·노출수 ↑
  - 메타데이터(title/description/OG) 최적화 → CTR ↑
  - 코어 업데이트 대응 → 순위 방어 / CTR 기인 변동 분리 분석
- **공유·바이럴 기능**
  - OG 태그·공유 카드 → 공유 링크 CTR ↑
  - 딥링크 / 유니버설 링크 → 외부 유입 → 앱 전환율 ↑
- **랜딩·리다이렉트**
  - 리다이렉트 페이지 정상화 → 유입 손실 차단
  - 랜딩 페이지 개선 → 유입 → 가입 전환율 ↑

> **매핑 예시**: "Event schema 적용 → 검색 노출 키워드 X개 확보, 오가닉 유입 X% ↑"

## 3. 제품 경험 품질 (성능·안정성·접근성·UX)

만든 산출물 자체의 비기능적 품질. 정량화가 쉬워 성과로 쓰기 좋다.

- **성능 (Performance)**
  - Core Web Vitals: LCP / INP / CLS 개선
  - 초기 로딩: FCP / TTI 단축
  - 번들 사이즈 ↓ (코드 스플리팅, lazy load, tree-shaking)
  - Lighthouse 점수 ↑
- **안정성 (Stability)**
  - 에러율 / 크래시율 ↓ (Sentry 등 모니터링)
  - 크로스 브라우저·디바이스 이슈 해결
  - 엣지 케이스 / 네트워크 예외 처리
- **접근성 (a11y)**
  - WCAG 준수, 스크린리더·키보드 내비게이션 지원
  - 색 대비 / 포커스 / aria 정비
- **UX 품질**
  - 인터랙션·애니메이션, 반응형·모바일 최적화
  - 디자인 일관성, 빈 상태/로딩/에러 상태 정교화

> **인과로 엮기(강력)**: "LCP 3.2s → 1.8s → 모바일 이탈률 X% ↓ → 세션당 페이지뷰 Y ↑"
> (이때 이탈률·매출 변화는 1번으로 크로스 참조)

---

## 4. 데이터·계측 (Analytics & Instrumentation)

숫자를 _만드는_ 게 아니라 숫자를 _알게_ 만드는 메타 역량. 위 1~3번을 측정 가능하게 만든 토대.

- **이벤트 설계·트래킹**
  - GA4 기반 이벤트 설계 / 사용자 행동 트래킹 구축
  - UTM 택소노미 수립 → 채널별 유입 정량화
  - 퍼널 단계별 이벤트 → 이탈 지점 가시화
- **어트리뷰션**
  - AppsFlyer vs 네이티브 어트리뷰션 평가·도입
  - 채널별 기여(CAC, 전환) 측정 기반 마련
- **실험 인프라**
  - A/B 테스트 구조 / 플래그 시스템
  - 지표 정의·대시보드 구축 → 의사결정 속도 ↑
- **데이터 정합성**
  - 중복·누락 이벤트 정리 → 지표 신뢰도 ↑

> **매핑 예시**: "GA4 + UTM 택소노미 구축 → 채널별 전환 측정 가능 → 마케팅 예산 배분 근거 제공"

---

## 5. 엔지니어링 시스템·DX (팀 단위 성과)

다음 개발을 빠르게 만드는 구조적 기여. composable·재사용 지향 개발자에게 가장 잘 맞는 영역.

- **재사용 자산**
  - 공통 컴포넌트 / 디자인 시스템 구축 → 신규 화면 개발 시간 ↓
  - 재사용 가능한 훅·유틸 → 중복 코드 제거
- **상태·데이터 패턴**
  - React Query 기반 쿼리 주도 상태관리 패턴 정립 → 중복 로직·보일러플레이트 ↓
  - 서버 상태 / 클라이언트 상태 분리 → 버그·복잡도 ↓
- **아키텍처**
  - composable·선언적 구조 도입 → 유지보수 비용 ↓
  - 폴더·모듈 경계 정리 → 온보딩·탐색성 ↑
- **개발 파이프라인**
  - 빌드 / CI 개선 → 배포 시간 ↓
  - 타입·린트·테스트 정비 → 회귀 버그 ↓

> **매핑 예시**: "쿼리 주도 패턴 도입 후 유사 기능 구현 N일 → M일, PR 리뷰 코멘트 수 ↓"

---

## 6. 조직·협업·지식 (사람·팀)

혼자 개발하더라도 명확한 성과가 되는 영역.

- **채용·평가**
  - 프론트 면접 진행
  - 질문 세트 / 평가 루브릭 설계 (신호·경고 기준)
- **지식·문서화**
  - 온보딩 문서 / 기술 결정 기록
  - 컨벤션·가이드 정립
- **협업 프로세스**
  - 디자이너·백엔드 협업 흐름 개선 (핸드오프, API 스펙 정렬)
  - 코드 리뷰 문화 / PR 템플릿
- **공유·멘토링**
  - 사내 공유 세션 / 외부 콘텐츠(빌드 인 퍼블릭)

> **매핑 예시**: "프론트 면접 질문 세트 + 루브릭 설계 → 평가 일관성 확보, 면접관 간 편차 ↓"

## (보조) 리스크·컴플라이언스

독립 카테고리로 두기엔 얇지만 누락 방지용으로 명시. 해당 작업이 있으면 위 1~6에 매핑하거나 별도 표기.

- 개인정보 / 권한 처리, 저작권 검토
- 보안 취약점(XSS 등) 대응
- 약관·고지 UI, 접근 권한 처리

## 작성 템플릿 (복붙용)

```
[카테고리] / [작업명]
- 문제: (Before 상태 + 그게 만든 손실)
- 행동: (내가 한 기술적 작업, 1~2줄)
- 결과: (지표 Before → After + 그게 만든 비즈니스/팀 결과)
- 근거: (대시보드/로그/PR 링크 — 검증 가능하게)
```

**예시**

```
[2. 유입] / 스케줄 페이지 Event schema 적용
- 문제: 일정 페이지가 일반 결과로만 노출돼 CTR 정체
- 행동: Event 구조화 데이터 마크업 + URL/slug 구조 개선
- 결과: 리치 결과 노출 → 노출 키워드 X개 확보, 오가닉 유입 +X%
- 근거: GSC 노출/클릭 추이, 색인 리포트
```
