# 강점 기반 배치 · 성과 정의 방법론

개인의 **강점**을 **직무 배치**와 **측정 가능한 성과**로 잇기 위한 방법론 모음입니다.
특정 팀·개인 데이터(면담·평가·내부 전략)는 포함하지 않으며, **재사용 가능한 절차와 원칙**만 공개합니다.

> 이 저장소는 "어떤 데이터로, 어떤 원칙에 따라" 강점을 공헌으로 연결하는지를 다룹니다.
> 프로젝트가 바뀌어도 절차는 그대로 두고 **데이터 소스만 갈아 끼웁니다.**

## 🎯 핵심 원칙

1. **강점 → 직무 → 공헌**: 강점 진단에서 멈추지 않고 측정 가능한 성과까지 연결한다.
2. **교차검증**: `재능 신호(갤럽) ↔ 실제 업무 증거 ↔ 자기보고(면담)`를 한 줄씩 대응시켜 분별한다.
3. **활동량 ≠ 성과**: 작업 "건수"는 도메인 친숙도 신호일 뿐, 숫자를 움직인 것만 성과로 본다.
4. **잠정 배치 → 계측 → 재평가**: 강점 기반 배치는 확정이 아니라 가설이며, 계측 후 실적으로 재배치한다.

## 🚶 빠른 시작 — 파트원 1명 강점 분석 (처음이라면 여기부터)

이 방법론을 처음 쓰는 사람이 **한 명을 분석**하는 최소 경로다. 자세한 원리는 [`guide.md`](guide.md) 참고.

### 0. 준비물 (데이터 소스)

| 소스 | 무엇을 말하나 | 필수? |
| --- | --- | --- |
| **면담 기록** (1:1 노트) | 동기·몰입·회피·자기인식 | ✅ 필수 |
| **업무 이력** (작업 로그/PR/티켓) | 실제 활동·도메인 친숙도 | ✅ 필수 |
| **갤럽 CliftonStrengths 34** | 타고난 재능·동기 | ⬜ **선택** — 있으면 1차 신호로, 없으면 생략 |
| _(있으면)_ **실적 지표(KR)** | 드러난 성과(숫자) | ⬜ 선택 — 보통 계측 후 단계 |

> **갤럽이 없으면** 면담 + 업무이력 **2소스**로 진행한다. 단, 재능 신호가 약하므로 **"신뢰도 낮음(잠정)"**을 명시한다.

### 단계 (5스텝)

1. **수집** — 면담 노트와 업무 이력을 모은다. (갤럽 있으면 상위/하위 테마도 뽑아 둔다)
2. **교차표 작성** — 아래 템플릿에 한 행씩 채운다. **2~3소스가 같은 곳을 가리키면** 그게 강점이다.
3. **강점·약점 도출** — 상위 신호가 몰린 도메인 = **엔진(강점)**, 하위에 몰린 도메인 = **약점**.
4. **직무를 강점에 맞춘다** — 강점 축에 책임을 모으고 **성과마다 단독 오너 1명**을 둔다. 약점 영역 일은 빼거나 페어·체크리스트로 **구조가 덮는다**(사람 탓 X).
5. **잠정 배치 → 재평가 기한 명시** — 활동 "건수"로 우열 금지. **언제 실적(KR)으로 다시 볼지** 날짜를 적는다.

### 교차표 템플릿 (복사해서 채우기)

```
| 후보 강점 | 갤럽 테마(선택) | 업무 증거 | 면담 발언 | 일치? | 배치 함의 |
| --- | --- | --- | --- | --- | --- |
| 예) 완성도/Craft | (있으면 기재) | UI 디테일 PR 다수 | "마감 품질이 신경 쓰인다" | ✅ 2~3소스 일치 | 전환 UX 오너 |
| 예) 실행 설계 | (없음) | 설계 작업 지연 이력 | "큰 그림보다 정해진 걸 선호" | ⚠️ 충돌 → 플래그 | 설계는 분리, 운영만 |
```

> **일치(✅)**면 강점 확정, **충돌(⚠️)**이면 덮어쓰지 말고 **플래그**로 남겨 다음 재평가의 입력으로 둔다.

### 끝내기 전 체크 (요약)

- [ ] 2소스(갤럽 있으면 3소스)를 교차표로 대응시켰나
- [ ] 강점 → 직무 → **측정 지표**까지 사슬이 이어지나
- [ ] 약점은 **구조**로 덮었나 / 성과마다 **단독 오너**인가
- [ ] 활동량(건수)으로 우열 매기지 않았나 / **재평가 기한**을 적었나

> 전체 체크리스트와 함정(소스별 오독 주의)은 [`guide.md`](guide.md) §5·§7 참고.
