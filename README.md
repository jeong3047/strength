# 강점 기반 배치 · 성과 정의 방법론

개인의 **강점**을 **직무 배치**와 **측정 가능한 성과**로 잇기 위한 방법론 모음입니다.
특정 팀·개인 데이터(면담·평가·내부 전략)는 포함하지 않으며, **재사용 가능한 절차와 원칙**만 공개합니다.

> 이 저장소는 "어떤 데이터로, 어떤 원칙에 따라" 강점을 공헌으로 연결하는지를 다룹니다.
> 프로젝트가 바뀌어도 절차는 그대로 두고 **데이터 소스만 갈아 끼웁니다.**

## 🎯 핵심 원칙

1. **강점 → 직무 → 공헌**: 강점 진단에서 멈추지 않고 측정 가능한 성과까지 연결한다.
2. **교차검증**: `재능 신호(갤럽) ↔ 실제 업무 증거 ↔ 자기보고(면담)`를 한 줄씩 대응시켜 분별한다.
3. **활동량 ≠ 성과**: 작업 "건수"는 도메인 친숙도 신호일 뿐, 숫자를 움직인 것만 성과로 본다.
4. **잠정 배치 → 계측 → 재평가**: 강점 기반 배치는 확정이 아니라 가설이며, 계측 후 실적으로 재배치한다.
